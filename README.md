# BukuSaku — Personal Expense Tracker

A fullstack personal finance tracking application with multi-currency support (IDR/USD), budget alerts, and real-time notifications. Built as a portfolio project to demonstrate fullstack development skills with a production-ready architecture.

## 🚀 Tech Stack

**Frontend**

- Next.js 14 (App Router)
- TypeScript
- Tailwind CSS v4
- shadcn/ui
- NextAuth.js (Google OAuth + Credentials)

**Backend**

- Express.js
- TypeScript
- PostgreSQL
- Prisma ORM v7
- Swagger (OpenAPI 3.0)

## 🏗️ Architecture

This project uses **NextAuth as the single source of authentication**. The Express backend does not issue its own session tokens — it only verifies the JWT generated by NextAuth using a shared secret (`NEXTAUTH_SECRET`), sent via the `Authorization: Bearer <token>` header.

This architectural decision was made after discovering that the initial approach (Express issuing a separate JWT cookie) caused the cookie to never reach the browser, since NextAuth's `authorize()` callback runs as a server-to-server call. The full analysis and resolution are documented in [`docs/BUGS.md`](./docs/BUGS.md).

```
Browser → NextAuth (session + backendToken)
              │
              ▼
    Header: Authorization Bearer <token>
              │
              ▼
    Express (verified via NEXTAUTH_SECRET)
```

## 🛠️ Development Setup

### Prerequisites

- Node.js 18+
- PostgreSQL

### Backend

```bash
cd backend
npm install
cp .env.example .env   # fill in your environment values
npx prisma migrate dev
npx prisma generate
npm run dev
```

### Frontend

```bash
cd frontend
npm install
cp .env.local.example .env.local   # fill in your environment values
npm run dev
```

The app runs at `http://localhost:3000`, and the API docs at `http://localhost:5000/api-docs`.
